为了深入理解 javascript，自己实现 new 的过程

先来看看官方 new 的定义

  1. Creates a blank, plain JavaScript object. For convenience, let’s call it newInstance.

  2. Points newInstance’s [[Prototype]] to the constructor function’s prototype property,
    if the prototype is an Object. Otherwise, newInstance stays as a plain object with Object.prototype as its [[Prototype]].
    Note: Properties/objects added to the constructor function’s prototype property are therefore accessible to all instances created from the constructor function.

  3. Executes the constructor function with the given arguments,
    binding newInstance as the this context (i.e. all references to this in the constructor function now refer to newInstance).

  4. If the constructor function returns a non-primitive, this return value becomes the result of the whole new expression. Otherwise,
    if the constructor function doesn’t return anything or returns a primitive, newInstance is returned instead.
    (Normally constructors don’t return a value, but they can choose to do so to override the normal object creation process.)

按照官方的定义一步步实现

实现一

  1. 定义一个空对象

  2. 将这个空对象的[[Prototype]] 指向函数（调用我的函数）的原型对象。

    [[Prototype]] 是一个引用，指向构造函数的原型对象，任何对象都有这个属性。

  3. 通过 call 改变 this，调用函数

  4. 返回函数执行结果

实现一、存在的问题

当构造函数返回的是一个对象时，不支持。
根据 mdn 定义，当 new 中的构造函数返回一个对象时，就返回这个对象。否则返回实例
